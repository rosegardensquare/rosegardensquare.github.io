Java 基本数据类型

1：基本数据类型的分类

  • byte

    -128 - 127 ，占1字节;

    1 个 byte 等于 8 个 bit

  • short

    占 2 字节

  • char

    占 2 字节，unicode 字符集

  • int

    -2 ^ 31 - 2 ^ 31-1, 大约 21 亿，占 4 字节

  • float

    4 字节；单精度，小数点之后可以精确到7 位；

    定义 float 时，结尾要加 f , 啥也不加就是 double 类型了

  • double

    8 字节；双精度，它的精度是 float 的 2 倍；

    写一个小数，后面啥也不加，默认数据类型是 double 类型, 当然，也可以加一个 d, 表示是 double 类型；

    浮点数都有舍入误差，例如：0.1 + 0.2 ≠ 0.3

  • long

    8 字节；定义 long 类型，后面需要加上 L

  • boolean

    4 字节；默认是 false

2：基本数据类型之间的运算

  1. byte、short、char 三个之间或者自己和自己做运算时，会首先把各自的类型提升为 int

  2. long l = 123;

123 默认是 int 类型，它自动提升为 long 类型

  1. long l = 123d;

123d 是 double 类型，double 类型赋值给 long 类型，double 会自动转为 long 类型

  1. long l = 23 亿；

编译报错，23 亿超出 int 的最大值范围

  1. int i = (int)0.5；

编译不报错，这里做了一个强制类型转换，将 double 类型强转为 int 类型，会丢失精度

4：金融计算使用 BigDecimal 代替浮点

  1. 比较两个 Bigdecimal 的大小时候，不能使用 BigDecimal 的 equals 方法，要使用它的 CompareTo 方法，这是阿里巴巴开发手册上写的。更不能使用 == 比较，因为 Big 是对象。

原因是：equals 方法比较的是两个数字的整数部分和小数部分，不会忽略精度。假如 0.1 和 0.10 的两个 BigDecimal 作 equals 比较，结果是fasle

  1. 减法：sub

    乘法：mu

    除法：div

    ​ 需要指定保留几位

  2. Bigdecimal(0.1) 和 BigDecimal(“0.1”) 的区别

    它们的舍入误差不一样；0.1转化为 BigDecimal 时，得先转化为二进制之后是 0.1111111……；

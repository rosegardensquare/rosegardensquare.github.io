线程的基础东西

1：并发、并行

并发（Concurrent）:

​ 两个进程，交替使用同一个 CPU。CPU的处理速度很快，用户觉察不出它的停顿，看来其就像同时干很多事情，这其实是cpu时间片技术，并发完成的；

并行：

​ 是对一个操作系统有 1 个以上的 CPU 来说的；当一个cpu 执行一个进程，另一个CPU执行另一个进程，两个进程间互不干扰可以同时进行，成为并行。

2：进程、线程

进程：一个任务就是一个进程，比如：打开一个浏览器就是一个浏览器进程，打开一个记事本，就是一个记事本进程；

​ 进程是分配资源的基本单位；

线程：在一个进程内部，要同时干很多事，就需要同时运行多个子任务，这些子任务叫做线程；

​ 线程是执行任务的基本单位；线程是比进程更轻量级的执行单元；

同一个进程中的多个线程共享资源；

3：创建内核线程的4种方式

  • 继承 Thread 类

    重写 run 方法；

  • 实现 runnable 接口

    重写 run 方法, 需要将 runnable 提交到Trhread 中才能执行；

    将 Runnable 作为参数传入 Thread 构造方法；

  • 实现有返回值的callable

    重写 call 方法，跟runnable 一样，它也是一个任务，需要提交到 Thread 中才能执行

  • futureTask 创建线程

  • 线程池

    其实使用的是线程工厂创建；

    DefaultThreadFactory实现ThreadFactory接口，最终还是使用new thread当时创建线程

4：Runnable 和 Callable 区别

都可以用来创建线程；

实现Runnable 需要重写 run 方法；实现 callable 接口，需要重写 call 方法；

runnable 的 run 方法没有返回值；callable 的 call 方法返回 Object 类型；

5：FutureTask 实现类

1：Future 接口

代表一个异步执行的结果；执行完成通过 get 获取执行结果；get 方法会阻塞；如果想取消，调用 cancel 方法；

FutureTask 是 Future 接口的实现类；

6：线程的状态相关

1：线程的6种状态

  • 初始 new 状态

    新建后还没有调用 start 方法之前

  • runnable 运行状态

    ​ Java 线程中将 ready 就绪和运行 running 统称为运行状态；

    • 就绪

      线程被调用了 start 方法，并不是马上就执行，需要等待获取 cpu 时间片；

    • 运行

      就绪状态的线程获取到了 cpu 时间片，开始执行代码

  • blocked 阻塞

    线程阻塞于锁

    被 sync 修饰的同步代码，所有因为获取不到锁的线程，会进入 EntryList 队列；

    被 Lock 修饰的同步代码，所有因为获取不到锁的线程，会进入 WaitingSet 队列(没有获取reentrantLock 锁的线程，当被唤醒时，正序唤醒 -> 拿到队头节点的下一个节点 ）：

    Thread 线程封装为 Node 对象 =》AQS 维护 Waitingset队列

  • waiting 等待

  • time-waiting 超时等待

  • 线程执行完毕

3：为啥线程没有 running 状态

cpu 的时间片很短，大概在 10-20 毫秒左右；

也就是说 1s 之内，同一个线程有可能处于就绪状态或者运行状态，如果明确的定义下来，其实状态时不准的；

当我们看到线程时 running 状态，它有可能已经失去cpu时间；

因此，对于线程，我们只需要直到它当前有没有获取时间片就行，这其实就是 runnable 状态；

7：线程中的关键字及区别

1：run 和 start 的区别

创建好线程后，想要启动线程需要调用 start 方法。因此，start 方法是启动一个线程的入口；

start 方法底层加了sync 锁，底层通过 jvm 配合操作系统，调用的是操作系统的 start0 方法来启动线程；

2：sleep方法 和 wait方法 、waiting 状态、time-waiting状态的区别

  • 作用的地方不同

    sleep方法 可以在任何地方使用；

    wait 方法只能在（sync）同步方法和同步代码块中使用；

​ 为啥要在同步代码中执行：因为，wait 方法执行前必须先持有锁，调用对象的wait方法；

  • 是否释放锁

    wait 方法会释放锁

    sleep 方法不会释放锁；

  • 调用 wait 方法之后

    JVM 会把该线程放入等待队列 waitSet 队列

    会释放锁

3：notify 和 notifyAll 的区别

当一个线程wait 之后，必须等待其他线程调用 notify 或者 notify all 方法，才会从等待队列中移出‘；

使用 notify all，会使所有线程从wait 状态唤醒，进入锁的争夺队列中，notify 只能唤醒一个线程；

被唤醒的这些线程只是进入争夺队列，并不表示立即获得CPU执行权，只是表示它们可以竞争锁，只有当竞争到锁之后才能有机会

被CPU调度；

8：用户线程和守护线程

用户线程执行的是用户级别的线程；

守护线程执行的是后台任务；

​ 最典型的是 GC；

这俩区别是：

​ JVM在所有的用户线程执行完毕之后退出，不等守护线程是否执行完毕；

9：虚拟线程

虚拟线程是jdk 21 新特性之一；它也叫做协程；它是用户态线程；

虚拟线程不同于普通的线程，它不在对应操作系统中的一个轻量级进程，而是将多个虚拟线程映射到少量的操作系统线程中，通过有效的调度减少上下文切换；因为它的切换不需要操作系统的参数；那么它的切换时怎么实现的呢？它是在 jvm 层面做的一些保持和恢复线程的信息，这些成本也是很低的。

线程安全就是解决多线程之间共享变量的问题；这个很重要。

在处理这个问题的时候要满足三个原则：

  • 原子性
  • 可见性
  • 有序性

    它在栈中

  • 局部变量

    它在堆中

2：Java 中的共享变量有哪些

共享变量有：类变量、成员变量；

主要集中在方法区和栈中；

这些地方的共享变量需要考虑线程安全问题。

11：关于线程上下文切换

线程上下切换发生在一个 CPU在多个线程之间的操作；

上下文切换是指：CPU从一个线程切换到另一个线程时，需要保存当前线程的状态，以便于下次恢复这个线程的时候能正常执行；

上下文信息包括：

​ 当前这个线程的状态、程序计数器、寄存器、栈指针等；

上下文切换这个动作会降低CPU的执行效率，这个动作是操作系统级别的切换；

底层需要操作系统在用户态和内核态之间切换，为啥这样呢？因为，一个Java线程它是对应操作系统中的一个轻量级进程，线程的创建和启动等动作，都需要操作系统参与；

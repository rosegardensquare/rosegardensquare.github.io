1：对 static 的理解

  • static 修饰的变量叫做类变量或者静态变量，在内存中只有一份；它跟 new 无关，所有的实例共享一份

    • 加载时机：

      随着类的加载而加载；因为类只加载一次，因此静态也只加载一次

  • 普通的变量叫做实例变量，存在堆中，每个对象有各自的；

    • 加载时机：

      随着对象的创建而创建

2：静态调用关系

静态只能调用静态的

静态下不能使用 this 或者 super;

普通可以调任何；

3：类加载顺序

  • 父类静态属性、静态代码块
  • 子类静态属性、静态代码块
  • 父类普通代码块
  • 父类构造器（无参、有参）
  • 子类普通代码块
  • 子类构造器（无参、有参）

4：static 修饰的变量在内存中的位置

1： jdk 7 之前，不包含jdk7

方法区保存static 变量、字符串常量池、此时的方法区叫做永久代

2：jdk 7 开始

static 变量和字符串常量池移到了堆中

3： jdk 8 开始

方法区叫做元空间

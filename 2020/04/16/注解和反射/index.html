1: 聊一下Java反射机制

java 的反射机制是指：

​ 程序在运行过程当中，可以创建任意一个类的实例、获取类的内部信息；包括：字段、方法、构造器、注解等；

​ 这种动态的获取类的信息的机制叫做反射机制；

​ 反射被视为Java能够实现动态语言的关键，他让Java有了动态性；

2: 反射机制的原理

反射的原理：

​ Java 的程序是这样运行的：java 文件通过编译期编译成 class文件，之后加载到内存，jvm 生成每个类的 java.lang.class 对象；

​ 通过 class 对象，可以获取类的内部信息，并做后续操作；因此，Java 的反射关键是这个 class 对象。

​ 反射是在运行的时候才知道类的信息；

​ 反射是约定大于配置，配置大于硬编码；

3： 反射机制的优缺点

优点：

  • 增加灵活性

    例如：spring 中利用配置解析 bean 的过程，就是利用反射的典型应用场景；

    Sping MVC 的请求调用对应的方法；

    JDBC 的 class.forName() ；

  • 增加复用及拓展性

    动态代理中，使用动态生成的代理类来提升代码的复用性

缺点：

  • 容易出现运行时异常

    相比于显式的接口和方法，编译器能帮助我们进行类型检查；而反射是在运行时才知道类的信息

  • 性能不高

    在运行方法或者获取字段前，需要遍历才能找到

  • 安全问题

    反射可以随意更改字段及方法的访问权限

2: 注解篇

1： 注解的理解

Annotation;

简单来说，就是对你想要扩展的点打上标记；当真正扩展的时候就处理这些标记所在类的属性、方法。

常常跟反射、AOP 结合使用。

2：注解包含

1：内置注解

2：元注解

负责解释其他注解

  • @Target ：作用的地方, 作用在类、方法、字段上；

  • @Rentition：生命周期，是在源代码、编译期、还是运行期；

    一般自定义注解都设置为运行时

3：自定义注解

@interface 这个东西自动继承 Java的Annotation 接口

1: 作用在方法上

2：作用在多个地方，用数组形式

2： 注解使用场景

  • 编译期检查

    • override
    • @FunctionalInterface 这是 jdk 8 的新特性

  • 注解 + 切面 （AOP）

    通过自定义注解来标注需要被切面处理的类或者方法，然后在切面中对方法的执行过程进行干预，比如在执行前后做一些操作；

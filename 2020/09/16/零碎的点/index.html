零碎的点

1：值传递和引用传递

除了集合、数组、对象会改变最终的值，因为三者是引用传递，其余都不会改变，因为其余的都是值传递，你传的是值过去，里面

做啥操作，跟外面没关系；

2：& 和 &&

&& 有短路功能；

&& 的左边是fasle, 右边就不再执行了；

3：抽象类和接口的区别

1：接口

  • 接口的方法默认是 public 修饰；
  • 接口不能有构造方法；
  • 从设计层面来说：接口相当于一种规范；

2：抽象类

  • 一个类只能有一个父类；

  • 从设计层面来说：抽象类相当于一种模板设计；

  • 抽象类可以有构造方法；

  • 抽象类主要是为了复用；我们经常说的模板方法模式；

4 ：成员变量和局部变量的区别

成员变量

​ 属于类变量或者实例；

​ 如果是 sttaic 修饰就是类变量，如果没有static 修饰就是实例变量；

​ 成员变量随着类的创建而存在；

​ 成员变量会自动赋值默认值；

局部变量

​ 属于方法中定义的变量或者方法参数；对象存在堆内存，局部变量也存在栈内存

局部变量随着方法的调用结束而结束；

​ 局部变量不会自动赋值；

5 ： 静态方法和实例方法

静态只能访问静态的，不能访问实例方法；

实例方法没有这方面的限制；

6：final 关键字

能作用的地方

  • 类
  • 方法
  • 变量

7 ：深拷贝和浅拷贝

深拷贝是创建一个一模一样的新对象，新旧对象之间不共享任何内存，新对象的修改不会影响老对象；

浅拷贝，拷贝的是地址；

无论是深拷贝还是浅拷贝，都需要实现 cloneable 接口；并在 clone 方法中重写拷贝的逻辑

8：如何理解多态

多态的概念比较简单：简单来说就是同一个操作作用于不同的对象，可以有不同的解释，产生不同的结果；

如果按照这个理解，多态是运行期的一种状态， Java 的重写是多态的体现，不过也有人提出重写是静态多态的提现，这个问题在 StackOverflow 上有很多讨论，但我更倾向于动态多态；

要实现这个运行期的动态绑定，就要满足 3 个条件：

​ 1：有类的继承或者实现

​ 2：要有重写

​ 3：父类引用指向子类对象

9：重载和重写的区别

1：重写是运行期的概念，只是参数列表不同；重载是编译期的概念，发生在父子类之间

2：运行期绑定，就是说在运行的时候，根据变量的引用指向子类对象，来调用具体的方法；

​ 编译期绑定，就是说在编译期根据参数的不同来决定调用哪个方法；

10 ：零拷贝（待补充）

磁盘的读写速度跟内存相差 10 倍以上；

所以针对磁盘的优化技术有很多：

  • 零拷贝
  • 直接 IO
  • 异步 IO

这些优化的目的是为了提高磁盘的吞吐量；

另外 OS 内核的高偶素缓冲区，也可以有效减少磁盘的访问次数；

零拷贝是为了提高 I/O 的吞吐效率；

kafka 利用零拷贝；

Nginx 也利用零拷贝；

所以 kafuka 之所以这么快的原因之一；

1：new

2：反射

3：序列化和反序列化

4：拷贝（深浅）

实现 Cloneable 接口

5：Unsafe

它是 JVM 提供的底层的不安全的操作：包括创建对象；是在内存中直接分配对象的实例，并返回对象的引用；

枚举

1：枚举的特点

枚举其实就是一个语法糖，定义的 Enum 类型，反映到字节码文件中就是 final class XXX extends java.lang.Enum

定义的属性反映到字节码上是：public static final aa

  • 构造方法私有，保证了只读不写，针对只读不写，我们只提供 get 方法，不提供 set 方法；
  • 该类是 final 修饰、hashCode 方法也是 final 修饰、属性是 static final 修饰
  • 枚举其实继承了 Enum 类

2：枚举中的方法

  • toString 方法

  • hashcode 方法

    不能被重写，因为是 final 修饰

  • values 方法

    返回枚举中的所有对象，下一步可以遍历

  • valueOf 方法

    通过常量名字获取枚举对象

  • ordinal

    属性的索引

  • compareTo 方法

    调用 ordinal 对两个属性相减

3： 枚举的好处

  • 它是天然的 单例。线程安全。这是因为底层做了保证
  • 枚举的单例可以避免被反序列化破坏

4： 枚举的比较方式

  • 使用 equals 或者 == 都行
  • 枚举的 equals 底层也是 ==

5 ： 啥时候使用枚举

当一个对象，天生有一组预定义的值，而且这些值不会轻易改变，只读不改，这个时候就可以使用枚举。

1： 有了常量了为啥还要用枚举

你要是使用常量也能做到，但是常量使用时候，调用者可以传递任何值都行，你要是使用枚举，那调用者在传入其他任何值的时候会编译报错，这时就规范了调用者的传参。

6： 纯粹是零碎的点

只能重写枚举的 toString 方法，不能下重写 hahscode 方法，因为 hashcode 是 final 定义；

7：枚举与单例模式相关

1：为什么说枚举是最好的单例实现

effective java 这本书中说：

​ 枚举实现的单例是最佳实践；

​ 原因在于：

​ 1：写法简单；

​ 2：枚举类的单例是线程安全的；

​ 3：避免被反序列化破坏；

​ 4：枚举实现的单例避免反射破坏

2：为什么说枚举的单例是线程安全的

枚举类反编译的字节码文件中可以看到：

​ 该类是final 修饰继承 Enum 类；

​ 各个属性是 final static 修饰；

静态代码块中加载所有属性；

既然是 static 修饰的属性，在类加载过程中就会被初始化，然而 Java 类的加载和初始化过程都是线程安全的；

因此，枚举类天生是线程安全的；

3：枚举类如何解决反序列化破坏单例

1：反序列化如何破坏单例

从Enum的源码中看到，大部分的方法都是final修饰的，特别是clone、readObject、writeObject这三个方法

不能通过克隆、序列化和反序列化复制枚举

4： 枚举单例如何解决反射破坏

即使单例类的构造方法是私有的，但是还是可以通过反射来破坏它；

枚举的构造方法虽然也是私有的，但是枚举是在类加载的时候加载，而普通方式实现的单例是在运行期调用的时候加载，这是两者的关键区别；因此，枚举不会被反射破坏，而普通方式ide单例会被反射破坏。

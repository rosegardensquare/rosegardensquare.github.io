Set 接口

1： Set 接口总述

元素不重复

  • TreeSet 底层是TreeMap, 是红黑树实现；

    自动排好序；

    不能存 null，要不然没法比较大小；

    添加的元素都要实现排序

  • HashSet 是 HashMap 实现；

    无序，可以有一个null

2：Set 如何保证不重复元素

1：HashSet 如何保证元素不重复

其实也就是 HashMap 咋存元素的过程

2：TreeSet 如何保证元素不重复

TreeSet 底层是 TreeMap 的 keySet方法，TreeMap 底层是红黑树，红黑树是一种平衡二叉树；

它能保证任何一个左右子树的高度差不会超过较矮的那颗的一倍。

TreeMap 的排序是按照 key 来排序，元素在插入时候，根据 comparTo 方法比较大小，所以 TreeMap

中的元素要实现 Coparable 接口；

TreeSet 是根据 ComparTo 方法判断是否重复元素。

3： LinkedHashSet

不仅元素不重复，还能记录插入和查询顺序；

基于 LinkedHashMap; 哈希表+双链表

通过双向链表记录插入顺序；

4：啥是 BitSet

BitSet 是位集合；

位集合的底层结构是一个 bit 数组，如果第 n 位是1，则表示数组 n 在这个集合中；

比如：当调用 BitSet 的 set(10) ，业务的意思是把 10 放到这个集合中，内部的操作是把二进制的第 10 位置为 1，

表示 10 在这个集合中；

在Java 中，它的底层是一个 long 数组，它最大的好处是对于多个数字来说，降低了内存空间，正常情况下，要把所有的 int

类型的正整数，存起来，需要 8G内存，但是使用 bitSet 的话，只需要原来内存的 1/32;

BitSet 常用在哪

  • 现在有一千万个随机数，随机数的范围在 1 - 1亿之间，求出 1-1亿之间的数字，哪些没有在这批随机数中？

  • 将 N 个数字排序

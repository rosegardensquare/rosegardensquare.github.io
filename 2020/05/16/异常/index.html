异常

1：异常体系

1： Throwable 是所有异常类的父类；

  • Error 子类

    它是系统级别的错误；jvm 抛出的异常，程序处理不了。

    常见异常有：内存溢出、堆栈异常

  • Exception 子类

    • 运行时的异常。runtimeException 及其子类，只在运行期间，不受检查异常

      • 空指针

      • 类型转换异常

        Treeset 中，添加元素，必须是相同的元素类型

      • 运算异常

      • 数组下标越界

      • 格式化异常

      • 并发修改异常（迭代器中迭代的同时删除元素）

      • nosuchelementexception(使用 iterator 遍历集合时，调用 next() , 如果没有元素

      • rejectedExecutionException，线程池中的拒绝策略

    • 编译期异常，，也叫不受检查异常。也叫受检查异常。编译期间抛出这个异常需要处理。使用 try catch 或者 throws 处理。

      • IoException

        对于输入流，要求 File 类的对象对应的物理磁盘必须存在，否则报 FileNotFoundException

      • SQLException

      • ClassNotFoundException

2：如何在代码中打印异常

  • e.printStackTrace()

    能打印完整的堆栈信息、异常的类型和原因

  • e.toString

    只有异常的类型和原因

  • e.getMessage

    只有原因

  • e.getStackTrace()

    只有堆栈信息

一般是使用 guava 的 Throwables.getStackAsString(e)

3：如何做一个自定义异常

分为两步：

  • 写一个异常类继承运行时异常 RuntimeException 或者 Exception ;
  • 重写构造方法
  • 在方法内部 throw new 刚才自定义的异常

4 ：关于 finally

finally 通常是跟 try 语句块使用；

通常情况下，不管有没有出发异常，finally 一定会执行；

所以一般把资源的释放和业务日志的打印放在 finally 中，

1： 什么时候 finally 不会执行

  • 程序还没有进入 try 中，程序已经中断；

  • try 或者 catch 中使 jvm 退出的

  • try 中出现死循环

2： finally 执行顺序

如果 finally 中有 return , 那么最终返回的是 finally 中的结果

如果 finally 中没有return, try 或者 catch 中，谁有，就返回谁的结果；
